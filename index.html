Why Trepo?

Trepo was conceived because there was no good open source general purpose genealogical API. Any developer wanting to innovate had to start from scratch each time they wanted to add their own unique twist to a genealogy software product. There was also a lack of true public/private collaboration in the industry, as well as a strong trend of centralization and ceding ownership and control of data to larger organizations. Trepo was created to address these problems and provide a solid foundation for others to build on.

What is Trepo?

Trepo is an open source genealogy API designed to enable public/private collaboration.

Trepo's language agnostic specification allows it to run on any standard graph database in any language. And since it is open source and extensible, companies and developers can take advantage of Trepo and tailor it to their specific needs.

Graph Database

Trepo is built on top of a directed property graph. Tinkerpop has an excellent introduction to property graphs available here

Trepo can be built on any property graph that meets the following requirements:

  • Supports directed edges.
  • Allows properties to be set on nodes and edges.
  • Allows nodes and edges to be retrieved or looked up via an assigned id.

vGraph

A distributable versioned property graph.

By adding additional requirements to a standard property graph, vGraph gains the ability to be versionable in a distributed environment. Each element is assigned a immutable UUID and label, and all changes are tracked through commits. Different databases, or repositories, can be linked together through the use of boundary nodes.

vGraph specification

dCap

vGraph commits, or sets of changes, can be stored, queried, and retrieved using dCap. You can query for things like:

  • Give me a list of authors who modified this element.
  • Who else is working on what I'm working on?
  • Get me a list of all changes to a repository within this specific time frame.
dCap specification

pTree

Defines a schema for storing genealogical information in vGraph. It is extensible by design, and takes full advantage of vGraph's capabilities.

pTree specification

nPipes

Provides a cross-repository query and traversal framework specific to vGraph.

nPipes specification

REST API

The actual API specification for Trepo. It exposes the capabilities of vGraph, dCap, pTree, and nPIpes, as well as providing additional utilities to provide a complete genealogical API.

Beyond the standard endpoints declared in the specification, the API may be extended in various ways to accommodate custom functionality and data models.
